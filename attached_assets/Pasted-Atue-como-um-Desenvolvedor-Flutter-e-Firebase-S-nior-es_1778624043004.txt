Atue como um Desenvolvedor Flutter e Firebase Sênior especializado em Apps de Geolocalização. Gere o código completo de um MVP para um aplicativo de zeladoria urbana chamado 'Zeladoria Digital'.

1. Stack Técnica e Dependências:

    Use Flutter com as seguintes dependências no pubspec.yaml: google_maps_flutter, geolocator, cloud_firestore, firebase_core.

    Utilize uma estrutura de código organizada, separando a lógica de Firebase da interface (UI).

2. Interface do Usuário (UI):

    Tema: Estética Dark Mode profissional. Cor principal: #DEFF9A (Lima), Background: #1A1A1A.

    Tela Principal: Mapa do Google em tela cheia ocupando 100% do espaço.

    Botão de Ação: Um FloatingActionButton estilizado com o ícone de '+' e rótulo 'REPORTAR'.

    Modal de Denúncia: Ao clicar em 'REPORTAR', abra um showModalBottomSheet arredondado com:

        Um seletor (Dropdown) com as categorias: 'Buraco na Via', 'Iluminação Pública', 'Acúmulo de Lixo', 'Enchente/Drenagem'.

        Um campo de texto para 'Descrição'.

        Um botão 'Confirmar Denúncia' com carregamento (loading state).

3. Lógica de Geolocalização:

    Ao abrir o app, peça permissão de localização e centralize o mapa na posição atual do usuário usando o pacote geolocator.

    Ao reportar, capture automaticamente a latitude e longitude atuais do usuário para salvar no banco de dados.

4. Integração com Firebase Firestore:

    Coleção: denuncias.

    Campos: id (auto-gen), categoria (string), descricao (string), latitude (double), longitude (double), status (string, default: 'Pendente'), timestamp (serverTimestamp).

    Lógica de Marcadores (Markers): O app deve escutar em tempo real (snapshots) a coleção denuncias do Firestore e renderizar um Marker no mapa para cada registro encontrado. Use cores diferentes para os marcadores se possível (ex: Vermelho para pendente).

5. Considerações Finais:

    Adicione comentários claros explicando onde devo inserir minha API KEY do Google Maps.

    Crie uma função _submitReport() que valide se os campos não estão vazios antes de salvar no Firebase.

    Se o Firebase não estiver inicializado, o app deve exibir um erro amigável.
